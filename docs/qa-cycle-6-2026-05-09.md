## Findings

### P2 - Closed profile dropdown creates authenticated mobile horizontal overflow

- Routes: `/home`, `/jobs`, `/messages`, `/profile-hub`
- Viewport: `390x844`, authenticated
- Actual: `document.documentElement.scrollWidth` is `448` while `clientWidth` is `390`.
- Expected: authenticated mobile app shell should not create horizontal page scroll.
- Evidence: overflow elements resolve to the closed `ProfileMenu` dropdown: `absolute right-0 z-50 mt-2 w-72` in `src/components/layout/ProfileMenu.tsx`. The menu is visually closed with `opacity-0 scale-95 -translate-y-2 pointer-events-none`, but it still contributes to page width.
- Likely fix: hide/unmount the dropdown when closed, or position it so the closed/offscreen state cannot affect document scroll width.

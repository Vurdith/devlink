## Findings

### P2 - Fixed: Settings pages overflowed horizontally on mobile

Routes:

- `/settings`
- `/settings/security`

Observed on mobile:

```text
/settings document width: 749px, viewport width: 390px
/settings/security document width: 749px, viewport width: 390px
```

Expected:

Settings pages should remain within the mobile viewport with no sideways scroll.

Actual:

The original QA pass found the page overflowing horizontally by roughly `359px`. The largest offenders were the settings layout/sidebar/card region and the fixed mobile bottom nav, both measuring about `749px` wide in the mobile pass.

Fixed area:

- `src/app/(main)/settings/layout.tsx`
- `src/app/(main)/settings/_components/SettingsNav.tsx`
- `src/components/layout/MobileNav.tsx`

Follow-up verification:

```text
/settings document width: 390px, viewport width: 390px
/settings/security document width: 390px, viewport width: 390px
/settings/notifications document width: 390px, viewport width: 390px
```

### P3 - Discover LCP image warning

Route: `/discover`

Observed on desktop:

```text
Image with src "https://cdn.devlink.ink/uploads/457c9240-4de0-4ea4-89e5-0336c043755f.gif" was detected as the Largest Contentful Paint (LCP).
```

Expected:

Above-the-fold media should either be intentionally prioritized or avoid becoming an expensive LCP candidate.

Actual:

Next.js logs an LCP warning for Discover media. The page still rendered and search input accepted text.

### P3 - Auth logo aspect-ratio warning

Routes:

- `/login`
- `/register`
- `/profile-hub` redirecting to `/login`

Observed on desktop and mobile:

```text
Image with src "/logo/logo-purple.png" has either width or height modified, but not the other.
```

Expected:

The logo image should preserve aspect ratio without framework warnings.

Actual:

Next.js warns that CSS changes only one dimension without setting the other to `auto`.

### P3 - Smooth-scroll route-transition warning

Route:

- `/profile-hub` redirecting to `/login`

Observed on desktop and mobile:

```text
Detected `scroll-behavior: smooth` on the `<html>` element.
```

Expected:

Route transitions should not trigger the Next.js smooth-scroll warning.

Actual:

Next.js recommends adding `data-scroll-behavior="smooth"` to the `html` element if smooth scrolling is intentional.

### P3 - Local Upstash cache warning

Routes:

- `/home`
- `/discover`

Observed on desktop:

```text
[Cache] Disabling Upstash Redis temporarily due to connectivity error.
```

Expected:

Local cache fallback should either connect successfully or degrade quietly enough that QA can distinguish expected local noise from regressions.

Actual:

The app logged cache degradation warnings, but affected routes rendered normally.

## Findings

### P3 - Discover LCP image warning

Route: `/discover`

Observed on desktop:

```text
Image with src "https://cdn.devlink.ink/uploads/457c9240-4de0-4ea4-89e5-0336c043755f.gif" was detected as the Largest Contentful Paint (LCP).
```

Expected:

Above-the-fold profile/media imagery should either be intentionally prioritized or avoided as LCP-heavy media.

Actual:

Next.js logs an LCP image warning for above-the-fold Discover media.

Likely area:

Discover profile/card media rendering.

### P3 - Auth logo aspect-ratio warning

Routes:

- `/login`
- `/register`
- `/profile-hub` redirecting to `/login`
- `/complete-signup` redirecting to `/login`

Observed on desktop and mobile:

```text
Image with src "/logo/logo-purple.png" has either width or height modified, but not the other.
```

Expected:

The logo image should preserve aspect ratio without console warnings.

Actual:

Next.js warns that CSS changes only one dimension without setting the other to `auto`.

Likely area:

Auth layout/logo image styling.

### P3 - Smooth scroll route-transition warning

Routes:

- `/profile-hub` redirecting to `/login`
- `/complete-signup` redirecting to `/login`

Observed on desktop and mobile:

```text
Detected `scroll-behavior: smooth` on the `<html>` element.
```

Expected:

Route transitions should not trigger Next.js smooth-scroll warnings.

Actual:

Next.js recommends adding `data-scroll-behavior="smooth"` to the `html` element if smooth scrolling is intentional.

Likely area:

Root layout or global CSS.

### P3 - Local Upstash cache warning

Route: `/home`

Observed on desktop:

```text
[Cache] Disabling Upstash Redis temporarily due to connectivity error.
```

Expected:

Local development should either connect to Redis or degrade quietly without noisy route-level warnings.

Actual:

The app logs a cache degradation warning. The route still renders normally.

Likely area:

Cache/Redis client fallback logging.
